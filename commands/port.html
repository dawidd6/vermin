port

port subcommand used to forward one or more ports to the VM from the local host.

To forward the requests on the localhost on port 9200 to vm_01 on port 9200: 

$ vermin port vm_01 9200
Forwarding from vm_01:9200 -> localhost:9200

Connected. Press CTRL+C anytime to stop
vermin uses the same pattern of docker in port mapping, however it doesn't require a local port and in such case the local port will be the same as vm port.
Syntax
vermin cp <vm> [local port:]<vm port> [[local port:]<vm port>...]

where:
[local port:] and vm port can take a form of range like: 9090-9095 or a single port like 9090.

[local port:] is optional, and when omitted, it takes the same value of the <vm port>.

Examples

1 - Forward to vm port 4040 from local port 4040: 

$ vermin port vm_01 4040

2 - Forward to vm port 4040 from local port 40040: 

$ vermin port vm_01 40040:4040

3 - Forward to vm port 4040 from local port 40040 and to port 8080 from 8080: 

$ vermin port vm_01 40040:4040 8080

4 - Forward to vm port 4040 from local port 40040 and to ports in range (8080 to 8088) from range(8080 to 8088): 

$ vermin port vm_01 40040:4040 8080-8088

5 - Forward to vm port 4040 from local port 40040 and to ports in range (8080 to 8088) from range(9080 to 9088): 

$ vermin port vm_01 40040:4040 9080-9088:8080-8088
