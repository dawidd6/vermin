mount

mount subcommand used to mount local filesystem to a directory inside the VM, by default if the guest directory not specified, then it defaults to /vermin.

It depends on virtualbox-guest-utils being install inside the VM to work. currently Many Vagrant images and ubuntu/focal image are supported.

Unlike , mount will keep the files synchronized between the host and the VM.

You can use the mount command against a single VM as many times as you want and each time will override the mount directly of previous times.
Mounts are transient, which means it will to not persist VM restarts.

To mount the ~/Downloads directory to /vermin inside the VM: 

$ vermin mount vm_01 ~/Downloads

To mount the ~/MyHTMLApp directory to /var/www/html inside the VM: 

$ vermin mount vm_01 ~/MyHTMLApp:/var/www/html

You can change the mounted directory to ~/Desktop using the command: 

$ vermin mount vm_01 ~/Desktop
mount command removes all previous mounts on the same VM and do a new mount.
