ip

ip subcommand used to find the IP address of a VM

$ vermin ip vm_01
192.168.100.160
The ip command depends on the bridge adapter of the VM, in case you manually change the setting in the VirtualBox to other settings like NAT or host-only, most of the vermin functions will not work.
Many other vermin commands that interact with the VM remotely such as , , and others depend on the ip function to get the IP of the VM under the hood before connecting to the VM.

Most of the time you will not need to use the ip command directly. However, in some cases for example when you try to to a VM, it will shows an error like: Cannot establish connection, in such case you will need to purge the IP cache using -p|--purge flag: 

$ vermin ip vm_03 -p
192.168.100.63

You can use the ip command inside scripts, for example if your VM running nginx, you use curl as follows: curl $(vermin ip vm_01)

Global discovery

vermin can find the IP address of VMs not managed by itself, to do so you need to pass the -g|--global flag: 

$ vermin ip -g MyUbuntuLinuxVM
