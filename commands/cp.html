cp

cp subcommand allows you to copy files and folders between a VM and the local filesystem or between two VMs.

You can copy files and folders from a local filesystem to a VM and vise versa. and you can copy files between two VMs in both directions.

If you want to mount files from local filesystem inside a VM use instead.

To copy /etc/os-release from the inside the vm to ~/temp on the host:

$ vermin cp vm_02:/etc/os-release ~/temp
Syntax
vermin cp <source> <destination>

where:
<source> and <destination> takes the form of <vm_name>:/path/to/file/or/directory.

In case of the <source> or <destination> is the local host, then the "<vm_name>:" part is removed.

Examples

1 - Copy file.txt from the host to user's home directory inside the VM: 

$ vermin cp ~/file.txt vm_01:~

2 - Copy file.txt from user's home directory inside the VM to the current directory on the host: 

$ vermin cp vm_01:project/file.txt .

3 - Copy file.txt from vm_01 home dir to the vm_02 /tmp dir: 

$ vermin cp vm_01:~/file.txt vm_02:/tmp
